# 移除开花阶段更新说明

## 更新日期
2026年4月9日

## 更新内容

### 1. 阶段调整

**原有阶段（8个）**：
1. 种子 (Seed) - 0%
2. 发芽 (Sprout) - 10%
3. 幼苗 (Seedling) - 25%
4. 小树 (YoungTree) - 50%
5. 成树 (MatureTree) - 75%
6. ~~开花 (Flowering) - 90%~~ ❌ 已删除
7. 结果 (Fruiting) - 95% → 调整为 90%
8. 成熟 (Harvest) - 100%

**更新后阶段（7个）**：
1. 种子 (Seed) - 0%
2. 发芽 (Sprout) - 10%
3. 幼苗 (Seedling) - 25%
4. 小树 (YoungTree) - 50%
5. 成树 (MatureTree) - 75%
6. 结果 (Fruiting) - 90% ⬅️ 阈值从95%调整为90%
7. 成熟 (Harvest) - 100%

### 2. 代码修改

#### OrangeTreeStage.cs
- 移除了 `Flowering = 5` 枚举值
- 调整了后续枚举值的索引：
  - `Fruiting`: 6 → 5
  - `Harvest`: 7 → 6

#### OrangeTreeController.cs
- 更新了默认阶段初始化，移除开花阶段
- 调整了结果阶段的阈值：95% → 90%

#### DeathModelConfigurator.cs
- 移除了开花阶段的死亡模型配置
- 更新了死亡模型名称映射

### 4. 生长时间调整（30秒总时长）

| 阶段 | 阈值 | 到达时间 | 持续时间 | 变化 |
|------|------|---------|---------|------|
| 种子 | 0% | 0秒 | 3秒 | 无变化 |
| 发芽 | 10% | 3秒 | 4.5秒 | 无变化 |
| 幼苗 | 25% | 7.5秒 | 7.5秒 | 无变化 |
| 小树 | 50% | 15秒 | 7.5秒 | 无变化 |
| 成树 | 75% | 22.5秒 | 4.5秒 | 无变化 |
| 结果 | 90% | 27秒 | 3秒 | ⬆️ 从1.5秒增加到3秒 |
| 成熟 | 100% | 30秒 | - | 无变化 |

**主要变化**：
- 移除了开花阶段（原本在27-28.5秒）
- 结果阶段的持续时间从1.5秒增加到3秒
- 总生长时间保持30秒不变

## 需要注意的事项

### 1. Unity场景配置

在Unity编辑器中，需要手动调整 `OrangeTree` 对象的 `Stages` 配置：

1. 选择 `OrangeTree` 对象
2. 在 Inspector 中找到 `Orange Tree Controller (Script)`
3. 展开 `Stages` 列表
4. **删除** Stage 5（开花阶段）
5. 调整 Stage 6（结果）的阈值：95 → 90
6. 使用"自动配置死亡模型"按钮重新配置

### 2. 已删除的模型

以下模型已从场景中移除（如果存在）：
- `OrangeTree_Flowering`
- `Flowering_Died`

### 3. 兼容性

如果你有保存的场景或预制体引用了开花阶段，需要手动更新：
- 检查所有引用 `OrangeTreeStage.Flowering` 的代码
- 更新场景中的阶段配置
- 重新配置死亡模型

   - 验证从成树（75%）直接跳转到结果（90%）

## 回滚方法

如果需要恢复开花阶段，执行以下步骤：

1. 在 `OrangeTreeStage.cs` 中恢复枚举：
```csharp
Flowering = 5,   // 开花
Fruiting = 6,    // 结果
Harvest = 7      // 成熟
```

2. 在 `OrangeTreeController.cs` 中恢复阶段初始化：
```csharp
stages.Add(new StageData(OrangeTreeStage.Flowering, "开花", 90f));
stages.Add(new StageData(OrangeTreeStage.Fruiting, "结果", 95f));
```

3. 在 `DeathModelConfigurator.cs` 中恢复死亡模型映射

4. 重新导入开花模型到场景中

## 总结

此次更新简化了橘子树的生长流程，移除了开花阶段，使得生长过程更加流畅。结果阶段的持续时间增加，让玩家有更多时间观察结果状态。总体生长时间保持30秒不变。
