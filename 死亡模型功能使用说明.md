# 橘子树死亡模型功能使用说明

## 功能概述

当温度或湿度**任意一个**不在推荐范围内时，橘子树会自动切换到对应阶段的死亡模型。当环境恢复正常时，会自动切换回正常模型并继续生长。

## 环境要求

根据 `OrangeTreeConfig` 配置：
- **温度范围**: 15°C - 35°C（最佳：25°C，容差：±10°C）
- **湿度范围**: 40% - 80%（最佳：60%，容差：±20%）

**注意**: 只要温度或湿度有一个超出范围，树木就会死亡！

## 配置步骤

### 方法一：使用自动配置按钮（推荐）

1. 在 Hierarchy 中选择 `OrangeTree` 对象
2. 在 Inspector 中找到 `Orange Tree Controller (Script)` 组件
3. 滚动到底部，点击 **"自动配置死亡模型"** 按钮
4. 等待配置完成的提示对话框

### 方法二：手动配置

在 Inspector 的 `Stages` 列表中，为每个阶段手动配置：

| 阶段 | Stage Model | Died Model |
|------|-------------|------------|
| 0 - 种子 | OrangeTree_Seed | 无（种子不会死亡） |
| 1 - 发芽 | OrangeTree_Sprout | Sprout_died |
| 2 - 幼苗 | OrangeTree_Seeding | Seeding_Died |
| 3 - 小树 | OrangeTree_YoungTreet | YoungTreet_Died |
| 4 - 成树 | OrangeTree_MatureTree | MatureTree_Died |
| 5 - 结果 | OrangeTree_Fruting | Fruting_Died |
| 6 - 成熟 | OrangeTree_Harvest | Harvest_Died |

## 工作原理

### 1. 环境监测
系统每帧检查：
- 温度是否在 [15°C, 35°C] 范围内
- 湿度是否在 [40%, 80%] 范围内

### 2. 死亡触发条件
- 温度 < 15°C **或** 温度 > 35°C
- **或** 湿度 < 40% **或** 湿度 > 80%
- 种子阶段（Stage 0）不会死亡

### 3. 模型切换
- **进入死亡状态**: 自动切换到当前阶段的 `diedModel`
- **恢复正常**: 自动切换回当前阶段的 `stageModel`
- 切换过程有 0.3 秒的淡入淡出效果

### 4. 生长控制
- **死亡状态**: 树木停止生长，生长值不再增加
- **恢复后**: 树木从当前生长值继续生长

## 测试方法

### 测试死亡功能

1. 运行游戏
2. 点击"开始生长"按钮
3. 等待树木生长到某个阶段（如幼苗、小树等）
4. 调整环境参数到危险范围：
   - **温度过低**: 调整到 < 15°C
   - **温度过高**: 调整到 > 35°C
   - **湿度过低**: 调整到 < 40%
   - **湿度过高**: 调整到 > 80%
5. 观察树木是否立即切换到死亡模型
6. 查看 Console 是否输出：`💀 树木死亡 - 当前阶段: XXX`

### 测试恢复功能

1. 在树木死亡状态下
2. 将环境参数调整回正常范围：
   - 温度: 15°C - 35°C
   - 湿度: 40% - 80%
3. 观察树木是否切换回正常模型
4. 查看 Console 是否输出：`✅ 树木恢复 - 当前阶段: XXX`
5. 确认树木继续生长

## 调试信息

系统会在 Console 中输出以下信息：

- `💀 树木死亡 - 当前阶段: XXX`: 树木进入死亡状态
- `✅ 树木恢复 - 当前阶段: XXX`: 树木恢复正常状态
- `✅ 已配置 XXX 的死亡模型: XXX`: 配置成功
- `⚠️ 未找到 XXX 的死亡模型: XXX`: 配置失败（模型不存在）

## 常见问题

### Q: 为什么树木不会死亡？
A: 检查以下几点：
1. 确保已配置死亡模型（使用自动配置按钮）
2. 确保环境参数确实超出范围
3. 确保不是种子阶段（种子不会死亡）
4. 查看 Console 是否有错误信息

### Q: 死亡模型没有显示？
A: 检查：
1. 死亡模型是否在场景中（OrangeTreeGrowthing 下）
2. 死亡模型的名称是否正确
3. 是否已正确配置到对应阶段的 `diedModel` 字段

### Q: 树木死亡后无法恢复？
A: 确保：
1. 温度**和**湿度都在正常范围内
2. 检查 Console 是否有错误信息
3. 尝试重新开始游戏

### Q: 如何清除死亡模型配置？
A: 
1. 选择 OrangeTree 对象
2. 在 Inspector 底部点击 **"清除死亡模型配置"** 按钮

## 代码修改说明

本功能修改了以下文件：

1. **OrangeTreeController.cs**
   - 添加了 `isDead` 状态标记
   - 添加了 `IsEnvironmentSuitable()` 方法检查环境
   - 添加了 `SwitchToDeadOrAlive()` 方法切换死亡/存活状态
   - 修改了 `Update()` 方法，在死亡状态下停止生长

2. **OrangeTreeStage.cs**
   - 在 `StageData` 类中添加了 `diedModel` 字段

3. **DeathModelConfigurator.cs** (新增)
   - 自定义编辑器，提供自动配置按钮

## 扩展建议

如果需要更复杂的死亡机制，可以考虑：

1. **渐进式死亡**: 环境不适合时，树木逐渐枯萎而不是立即死亡
2. **不可逆死亡**: 死亡后无法恢复，需要重新种植
3. **死亡动画**: 添加从正常到死亡的过渡动画
4. **死亡原因提示**: 显示具体是温度还是湿度导致的死亡
5. **阶段特定容差**: 不同阶段对环境的容忍度不同

## 技术细节

- **检测频率**: 每帧检查一次环境条件
- **切换延迟**: 0.3 秒淡入淡出
- **性能影响**: 极小，只在状态变化时执行切换
- **保存状态**: 死亡状态不会保存，重新运行游戏会重置
