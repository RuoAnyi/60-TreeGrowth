# 收获提示和产量系统说明

## 功能概述

橘子树系统现在已经实现了完整的收获提示和产量惩罚机制：

### 1. 收获提示功能 ✅

当生长值达到 **100%** 时，系统会：
- 自动显示收获消息UI
- 提示文本：**"恭喜您！您已收获 X 公斤的橘子！"**
- X 为当前实际产量（受停止生长次数影响）
- 消息会自动在5秒后消失，或点击"知道了"按钮手动关闭

### 2. 产量惩罚机制 ✅

每次环境从**适宜变为不适宜**（生长停止）时：
- 产量减少 **1公斤**
- 初始最大产量：**50公斤**
- 最低产量：**0公斤**（不会变成负数）

### 3. 环境适宜条件

生长需要同时满足：
- **温度**：在推荐范围内（默认 15-35°C）
- **湿度**：在推荐范围内（默认 40-80%）

当温度或湿度超出范围时，生长停止，产量减少。

## 代码实现

### OrangeTreeController.cs

关键方法：

```csharp
// 生长停止时调用
private void OnGrowthStopped()
{
    currentYield = Mathf.Max(0, currentYield - yieldLossPerStop);
    OnYieldChanged?.Invoke(currentYield);
    Debug.Log($"⚠️ 生长停止！产量减少 {yieldLossPerStop}kg，当前产量: {currentYield}kg");
}

// 达到100%时调用
private void OnHarvestReached()
{
    hasHarvested = true;
    OnHarvestComplete?.Invoke(currentYield);
    Debug.Log($"🎉 恭喜您！您已收获 {currentYield} 公斤的橘子！");
    ShowHarvestMessage(currentYield);
}

// 显示收获消息
private void ShowHarvestMessage(int yield)
{
    HarvestMessageUI harvestUI = FindObjectOfType<HarvestMessageUI>();
    if (harvestUI != null)
    {
        harvestUI.ShowMessage(yield);
    }
}
```

### HarvestMessageUI.cs

收获消息UI控制器：
- 自动订阅 `OnHarvestComplete` 事件
- 显示收获消息面板
- 支持自动隐藏（5秒）或手动关闭
- 文本格式：`恭喜您！您已收获 {yield} 公斤的橘子！`

## 测试场景

### 测试1：正常生长到100%
1. 保持温度和湿度在适宜范围内
2. 等待生长到100%
3. 应该显示：**"恭喜您！您已收获 50 公斤的橘子！"**

### 测试2：中途停止生长
1. 开始生长
2. 调整温度或湿度到不适宜范围（停止生长1次）
3. 调回适宜范围继续生长
4. 重复步骤2-3多次
5. 达到100%时，产量应该是：**50 - 停止次数**

例如：
- 停止3次 → 收获 47公斤
- 停止10次 → 收获 40公斤
- 停止50次以上 → 收获 0公斤

## 事件系统

系统提供以下事件供其他组件订阅：

```csharp
public event Action<int> OnYieldChanged;      // 产量变化
public event Action<int> OnHarvestComplete;   // 收获完成
```

## 配置参数

在 Inspector 中可以配置：

**OrangeTreeController:**
- `maxYield`: 最大产量（默认50kg）
- `currentYield`: 当前产量
- `yieldLossPerStop`: 每次停止损失的产量（默认1kg）

**HarvestMessageUI:**
- `autoHideDuration`: 自动隐藏时间（默认5秒）
- `autoHide`: 是否自动隐藏（默认true）

## 注意事项

1. 产量只在**环境从适宜变为不适宜**时减少，持续不适宜不会重复减少
2. 收获消息只在**第一次达到100%**时显示一次
3. 重置生长会恢复产量到最大值（50kg）
4. 产量显示在左上角的 YieldDisplay UI 中实时更新
