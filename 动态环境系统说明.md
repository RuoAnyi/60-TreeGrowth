# 动态环境系统说明

## 功能概述

实现了一个动态环境系统，温度和湿度会自动快速变化。玩家需要通过点击上升/下降按钮来调节环境参数，保持在适宜果树生长的范围内。

## 系统特点

### 1. 自动动态变化
- **温度**: 在 0-100 范围内自动快速变化
- **湿度**: 在 0-100 范围内自动快速变化
- **变化速度**: 可配置，默认较快以增加游戏挑战性
- **方向改变**: 每隔一段时间随机改变变化方向

### 2. 玩家交互
- 通过点击**上升按钮**增加温度/湿度
- 通过点击**下降按钮**减少温度/湿度
- 需要持续关注并调节，保持环境适宜

### 3. 游戏机制
- **适宜范围**: 温度 30-70，湿度 40-80
- **超出范围**: 树木进入死亡状态，停止生长
- **恢复正常**: 调节回适宜范围后，树木恢复生长

## 配置参数

### EnvironmentManager 配置

在 Unity Inspector 中可以调整以下参数：

#### 动态变化设置
| 参数 | 默认值 | 说明 |
|------|--------|------|
| Enable Dynamic Change | true | 是否启用自动变化 |
| Temperature Change Speed | 8.0 | 温度变化速度（单位/秒） |
| Humidity Change Speed | 10.0 | 湿度变化速度（单位/秒） |
| Direction Change Interval | 3.0 | 方向改变间隔（秒） |

#### 变化范围
| 参数 | 默认值 | 说明 |
|------|--------|------|
| Temperature Range | (0, 100) | 温度变化范围（0-100） |
| Humidity Range | (0, 100) | 湿度变化范围（0-100） |

### EnvironmentController 配置

#### 调节步长
| 参数 | 默认值 | 说明 |
|------|--------|------|
| Temperature Step | 2.0 | 每次点击调节的温度变化量 |
| Humidity Step | 5.0 | 每次点击调节的湿度变化量 |

## 工作原理

### 1. 动态变化机制

```
每帧更新：
1. 温度 += 温度变化速度 × 方向 × Time.deltaTime
2. 湿度 += 湿度变化速度 × 方向 × Time.deltaTime
3. 到达边界时自动反转方向
4. 定期随机改变方向（增加不可预测性）
```

### 2. 方向控制

- **初始方向**: 游戏开始时随机
- **边界反转**: 到达最大/最小值时自动反转
- **随机改变**: 每隔 `directionChangeInterval` 秒，有50%概率改变方向

### 3. 玩家调节

```
点击上升按钮：
- 温度/湿度 += 调节步长
- 立即生效，不影响自动变化

点击下降按钮：
- 温度/湿度 -= 调节步长
- 立即生效，不影响自动变化
```

## 游戏平衡建议

### 难度调整

#### 简单模式
```
Temperature Change Speed: 1.5
Humidity Change Speed: 2.0
Direction Change Interval: 5.0
Temperature Step: 2.0
Humidity Step: 10.0
```

#### 普通模式（当前）
```
Temperature Change Speed: 3.0
Humidity Change Speed: 4.0
Direction Change Interval: 3.0
Temperature Step: 1.0
Humidity Step: 5.0
```

#### 困难模式
```
Temperature Change Speed: 5.0
Humidity Change Speed: 6.0
Direction Change Interval: 2.0
Temperature Step: 0.5
Humidity Step: 3.0
```

### 变化范围调整

#### 宽松范围（更容易保持在适宜区）
```
Temperature Range: (15, 35)  // 正好是适宜范围
Humidity Range: (40, 80)     // 正好是适宜范围
```

#### 标准范围（当前）
```
Temperature Range: (10, 40)  // 超出适宜范围
Humidity Range: (30, 90)     // 超出适宜范围
```

#### 极端范围（更具挑战性）
```
Temperature Range: (5, 45)   // 大幅超出适宜范围
Humidity Range: (20, 95)     // 大幅超出适宜范围
```

## 使用示例

### 场景配置

1. **选择 EnvironmentManager 对象**
2. **在 Inspector 中配置**：
   ```
   Enable Dynamic Change: ✓
   Temperature Change Speed: 3
   Humidity Change Speed: 4
   Direction Change Interval: 3
   Temperature Range: (10, 40)
   Humidity Range: (30, 90)
   ```

3. **运行游戏**：
   - 温度和湿度开始自动变化
   - 观察数值显示
   - 点击按钮调节

### 测试场景

#### 测试1：观察自动变化
1. 运行游戏但不点击任何按钮
2. 观察温度和湿度的变化
3. 注意树木何时进入死亡状态

#### 测试2：玩家调节
1. 等待环境参数进入危险区域
2. 快速点击按钮调节回适宜范围
3. 观察树木是否恢复生长

#### 测试3：持续挑战
1. 尝试在整个30秒生长期间保持环境适宜
2. 测试是否能成功让树木完成生长

## 视觉反馈建议

### 1. 颜色提示
- **绿色**: 环境适宜（15-35°C, 40-80%）
- **黄色**: 接近危险（12-15°C或35-38°C, 35-40%或80-85%）
- **红色**: 危险区域（<12°C或>38°C, <35%或>85%）

### 2. 动画效果
- 数值变化时添加缓动动画
- 危险时添加闪烁效果
- 树木死亡时添加警告提示

### 3. 音效建议
- 点击按钮：短促的点击音
- 进入危险区：警告音效
- 树木死亡：失败音效
- 成功生长：成功音效

## 代码示例

### 获取当前环境状态

```csharp
EnvironmentManager env = EnvironmentManager.Instance;
float temp = env.Temperature;
float humid = env.Humidity;

// 检查是否在适宜范围
bool isTempOk = temp >= 15f && temp <= 35f;
bool isHumidOk = humid >= 40f && humid <= 80f;
bool isEnvironmentSuitable = isTempOk && isHumidOk;
```

### 手动调节环境

```csharp
// 增加温度
env.AdjustTemperature(1f);

// 减少温度
env.AdjustTemperature(-1f);

// 增加湿度
env.AdjustHumidity(5f);

// 减少湿度
env.AdjustHumidity(-5f);
```

### 重置到最佳状态

```csharp
env.ResetToOptimal(); // 温度25°C，湿度60%
```

## 性能优化

- 环境变化每帧计算，性能开销极小
- 使用事件系统通知变化，避免轮询
- 只在值实际改变时触发更新

## 故障排除

### 问题1：环境不变化
**检查**：
- `Enable Dynamic Change` 是否勾选
- `Change Speed` 是否大于0

### 问题2：变化太快/太慢
**调整**：
- 修改 `Temperature/Humidity Change Speed`
- 建议范围：1-10

### 问题3：按钮无效
**检查**：
- EnvironmentController 是否正确绑定按钮
- EnvironmentManager 是否存在于场景中

### 问题4：树木一直死亡
**调整**：
- 降低变化速度
- 增加调节步长
- 扩大适宜范围（修改 GrowthConfig）
